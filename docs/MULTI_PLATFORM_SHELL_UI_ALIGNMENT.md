# Multi-Platform Shell UI Alignment (iOS/macOS/tvOS)

## 1. 背景

当前 iOS 已具备完整的“订阅源插件 + 默认壳 UI”流程：
- 无 sandbox 插件时，平台入口显示配置引导。
- 可输入 URL，自动识别视频链接或插件订阅源（`.json`）。
- 订阅源拉取后可安装/更新/全部安装插件。
- 安装完成后自动切换回完整平台 UI。

本次目标是让 macOS/tvOS 与 iOS 行为对齐。

---

## 2. iOS 现状（基线）

### 2.1 URL 输入判别
- 通过后缀判断是否为订阅源：
  - URL path extension 为 `json` 或字符串后缀 `.json` -> 订阅源。
  - 其他 -> 视频地址收藏。

### 2.2 订阅内容交互
- 添加订阅源后立即展示“订阅内容”弹层。
- 列表支持状态：加载中、空态、安装中、已安装、可更新、失败。
- 支持操作：安装、更新、全部安装。
- 错误显示：索引拉取错误与安装失败错误在 UI 可见，并输出日志。

### 2.3 默认 UI 切换
- `PluginAvailabilityService.hasAvailablePlugins == false`：显示壳配置页。
- 安装插件成功后刷新可用性与平台列表，自动回切完整 UI。

---

## 3. 对齐矩阵（目标）

| 能力 | iOS | macOS | tvOS | 说明 |
|---|---|---|---|---|
| 无插件默认进入配置 UI | 已完成 | 本次补齐 | 本次补齐 | 平台入口默认行为一致 |
| URL 自动识别视频/订阅 | 已完成 | 本次补齐 | 本次补齐 | `.json` 视作订阅源 |
| 订阅内容列表（安装/更新/全部安装） | 已完成 | 本次补齐 | 本次补齐 | 与 iOS 同策略 |
| 安装后刷新并回切完整平台 UI | 已完成 | 本次补齐 | 本次补齐 | 监听安装插件 ID 变化 |
| 配置页展示“已添加订阅源列表” | 不展示 | 不展示 | 不展示 | 全平台统一约束 |

---

## 4. 统一交互约束

1. 配置页不展示“已添加的订阅源”列表。
2. 订阅内容仅通过弹层/Sheet 呈现，并在列表内完成安装与更新。
3. 更新按钮仅在存在更高远端版本时显示。
4. 错误信息必须可见（UI）且可追踪（日志/控制台）。
5. 不修改 LiveParse 协议，不变更 LiveType 语义。

---

## 5. 平台实施要点

### 5.1 macOS
- 在根视图注入并维护：
  - `PluginAvailabilityService`
  - `PluginSourceManager`
  - `StreamBookmarkService`
- 启动时 `checkAvailability()`，并根据 `installedPluginIds` 刷新平台列表。
- 无插件时平台入口显示 `MacShellConfigView`。
- 搜索入口在无插件时隐藏（与 iOS 对齐）。

### 5.2 tvOS
- 在 `AppState` 增加：
  - `pluginAvailability`
  - `pluginSourceManager`
  - `bookmarkService`
- 启动时 `checkAvailability()`。
- 无插件时平台页显示 `TVShellConfigView`。
- 搜索入口在无插件时隐藏。

## 6. 验收清单

### 6.1 通用
- [ ] 无插件首次进入平台页，展示配置引导。
- [ ] 输入普通 URL 可添加到视频收藏。
- [ ] 输入 `.json` 可拉取订阅内容并展示插件项。
- [ ] 可单个安装、更新、全部安装。
- [ ] 安装成功后自动刷新平台列表并回切完整 UI。
- [ ] 异常场景有清晰错误提示（非法 URL、拉取失败、安装失败）。

### 6.2 macOS
- [ ] 窗口缩放下配置布局稳定。
- [ ] 订阅内容 Sheet 工具栏可用（完成/全部安装）。

### 6.3 tvOS
- [ ] 订阅配置页面焦点可达。
- [ ] 遥控器可完成输入、安装、返回。
